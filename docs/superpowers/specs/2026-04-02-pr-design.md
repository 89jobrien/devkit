# devkit pr — Design Spec

## Goal

`devkit pr` generates a structured pull request description from git diff, log, and stat — printed to stdout, ready to paste or pipe into `gh pr create`.

## Command Interface

```
devkit pr [--base <branch>]
```

| Flag | Default | Description |
|------|---------|-------------|
| `--base` | auto-detect | Base branch to diff against. Auto-detected via `gh repo view --json defaultBranch`, fallback `main`. |

## Output Format

Plain markdown printed to stdout, logged to `~/.dev-agents/<project>/ai-logs/` via `SaveCommitLog`.

```markdown
# <title>

<summary>

## Changes
- …

## Test Plan
- …
```

## Architecture

### BAML

New file `internal/baml/baml_src/pr.baml`:

```baml
class PRDescription {
  title string @description("Short PR title, ≤72 characters")
  summary string @description("2-4 sentence overview of what changed and why")
  changes string[] @description("Bullet list of key changes")
  test_plan string[] @description("Bullet list of how to verify the changes")
}

function DraftPR(git_context: string) -> PRDescription {
  client AnthropicBalanced
  prompt #"
    You are drafting a pull request description for a software engineer.

    {{ gitContext }}

    Write a clear, concise PR description. The title should be ≤72 characters.
    The summary should explain what changed and why in 2-4 sentences.
    Changes should be specific, actionable bullets.
    Test plan should describe concrete verification steps.

    {{ ctx.output_format }}
  "#
}
```

After adding `pr.baml`, regenerate the BAML client:
```
baml-cli generate
python3 internal/baml/patch-generated.sh
```

### New package: `internal/pr`

Single file `pr.go`:

```go
type Runner interface {
    Run(ctx context.Context, prompt string, tools []string) (string, error)
}
type RunnerFunc func(ctx context.Context, prompt string, tools []string) (string, error)

type Config struct {
    Base   string // resolved base branch
    Diff   string
    Log    string
    Stat   string
    Runner Runner
}

func Run(ctx context.Context, cfg Config) (string, error)
func ResolveBase(explicit string) string
```

`Run` builds a single prompt from the git context fields and calls `cfg.Runner.Run`. No parallelism.

`ResolveBase(explicit string) string`:
- If `explicit != ""`, return it unchanged
- Run `gh repo view --json defaultBranch --jq .defaultBranch`
- On any error (gh not installed, not a GitHub repo, no network), return `"main"`

**Prompt structure:**
```
You are generating a pull request description for a software engineer.

Base branch: <base>

Recent commits:
<log>

Changed files:
<stat>

Diff:
<diff (truncated to 8000 chars if needed)>

Produce a PR description with title, summary, list of changes, and test plan.
```

The `internal/baml` adapter gets a new `"pr"` role in `adapter.go`:
- Calls `baml_client.DraftPR(ctx, prompt)`
- Formats result as markdown via `formatPRDescription(r *types.PRDescription) string`

### Formatter

```
# <title>

<summary>

## Changes
- <change 1>
- <change 2>

```

### Wiring in `cmd/devkit/main.go`

```go
var prBase string
prCmd := &cobra.Command{
    Use:   "pr",
    Short: "Draft a pull request description from branch diff",
    RunE: func(cmd *cobra.Command, args []string) error { ... },
}
prCmd.Flags().StringVar(&prBase, "base", "", "Base branch (default: auto-detect from GitHub)")
```

Uses `baml.New("pr", os.Stdout)` if `cfg.Providers.UseBAML`, otherwise `router.For(providers.TierBalanced)`.

Logs result via `devlog.SaveCommitLog`.

## Error Handling

- `gh` not installed or not a GitHub repo: `ResolveBase` falls back to `"main"` silently
- No commits in range: include "no commits in window" in prompt — LLM handles gracefully
- Diff too large: truncate to 8000 chars with a `[diff truncated]` suffix before sending
- BAML not enabled: fall back to plain runner with the same prompt, LLM returns unstructured markdown

## Testing

- `internal/pr/pr_test.go` — stub runner, assert prompt contains diff and log
- Test `ResolveBase` fallback when `gh` fails (mock via `RunnerFunc` or by running against a non-GitHub dir)
- Test `formatPRDescription` with a known `PRDescription` struct
